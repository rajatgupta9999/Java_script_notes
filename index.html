Difference betwenn "var","let","const", Variables

  1. Scope:-
    1. var:- Function-scoped or global ex:- Visible within the entire function
    2. let:- Block-scoped (ES6+) ex:- Visible only within { } blocks
    3. Const:- Block-scoped (ES6+) ex:-Visible only within { } blocks
  2. Reassigned:-
    1. var:- Yes
    2. let:- No
    3. Const:- No (except for objects)
  3. Redeclaration:-
    1. var:- Yes(even in the same scope)
    2. let:- No
    3. Const:- No
  4. Hoisting:-
    1. var:- Hoisted and initialized to undefined.
    2. let:- Hoisted but not initialized (temporal dead zone).
    3. Const:- Hoisted but not initialized (temporal dead zone).
  5. Initialization:-
    1. var:- No (defaults to undefined).
    2. let:- No (defaults to undefined).
    3. Const:- Yes (must assign a value when declared).
  6. Use Cases:-
    1. var:- Avoid in modern code (legacy code only).
    2. let:- Use for variables that need to be reassigned (e.g., loop counters).
    3. Const:- Default choice for variables that shouldn’t change (prevents accidental reassignment).

Loops In JavaScript

- Loops are a way to repeat code until a condition is met
  • Types of Loops:-
    1. for Loop:- Runs a block of code a set number of times. You control the start, stop, and step.
      Syntax:- for (initialization; condition; increment) {code}.
    2. while Loop:- Executes code while a condition is true (good for unknown iterations).
      Syntax:- while (condition) {code}.
    3. do...while Loop:- Executes code at least once before checking the condition.
      Syntax:- do {code} while (condition);
    4. for...of Loop:- Loops over iterable objects (arrays, strings, etc.). Clean and modern.
      Syntax:- for (const element of iterable) {code}
    5. for...in Loop:- Loops over an object’s enumerable properties. Mostly for objects, not arrays.
      Syntax:- for (const key in object) {code}
      • break Exits the loop early.
      • continue Skips to the next iteration.
