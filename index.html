Difference betwenn "var","let","const", Variables

  1. Scope:-
    1. var:- Function-scoped or global ex:- Visible within the entire function
    2. let:- Block-scoped (ES6+) ex:- Visible only within { } blocks
    3. Const:- Block-scoped (ES6+) ex:-Visible only within { } blocks
  2. Reassigned:-
    1. var:- Yes
    2. let:- No
    3. Const:- No (except for objects)
  3. Redeclaration:-
    1. var:- Yes(even in the same scope)
    2. let:- No
    3. Const:- No
  4. Hoisting:-
    1. var:- Hoisted and initialized to undefined.
    2. let:- Hoisted but not initialized (temporal dead zone).
    3. Const:- Hoisted but not initialized (temporal dead zone).
  5. Initialization:-
    1. var:- No (defaults to undefined).
    2. let:- No (defaults to undefined).
    3. Const:- Yes (must assign a value when declared).
  6. Use Cases:-
    1. var:- Avoid in modern code (legacy code only).
    2. let:- Use for variables that need to be reassigned (e.g., loop counters).
    3. Const:- Default choice for variables that shouldn’t change (prevents accidental reassignment).

Loops In JavaScript

- Loops are a way to repeat code until a condition is met
  • Types of Loops:-
    1. for Loop:- Runs a block of code a set number of times. You control the start, stop, and step.
      Syntax:- for (initialization; condition; increment) {code}.
    2. while Loop:- Executes code while a condition is true (good for unknown iterations).
      Syntax:- while (condition) {code}.
    3. do...while Loop:- Executes code at least once before checking the condition.
      Syntax:- do {code} while (condition);
    4. for...of Loop:- Loops over iterable objects (arrays, strings, etc.). Clean and modern.
      Syntax:- for (const element of iterable) {code}
    5. for...in Loop:- Loops over an object’s enumerable properties. Mostly for objects, not arrays.
      Syntax:- for (const key in object) {code}

      • continue Skips to the next iteration.

Events In JS

-The change in the state of an object is known as an event.
-Events are fired to notify code of interesting changes that may affect code execution.
-JavaScript events are actions or occurrences (e.g., clicks, key presses, mouse movements) that happen in the browser and can be detected and handled using event listeners. They allow interactivity in web applications by triggering code when specific events occur on DOM elements.
concept:-
  1. Event:- A signal that something has happened (e.g., click, mouseover, keydown).
  2. Event Listener:- A function that "listens" for an event and executes when the event occurs.
  3. Event Handler:- The function or code that runs in response to the event.
  4. Event Object:- An object passed to the event handler containing details about the event (e.g., event.target, event.type).

  1. Event Handling

     - A function that runs when the event occurs.
    Syntax:- node.event()=>{
    //handle here
    }
    Ex:-let bta1= document.querySelector("btn1");
    btn1.onclick =() =>{
    console.log("Button have been clicked");
    };

  2. Event Object

     - Event Object is automatically created when an event occurs.this object contains details about the event and provides methods to control its behavior.it is argument to event handler function.
    Syntax:- node.event=(name the event here)=>{
    console.log(event name.type);
    }

  3. Event Listener

     - An event listener is a function or mechanism that "Listens" for a specific event on a Dom Element and trigger a callback function.
    - A binding between an event,an element,and a handle function.
    Syntax:- element.addEventListener(event.handler[options]);
    Ex:-btn1.addEventListener("mouseover",()=>{
    console.log("mouse is over the element");
    })

Popular Events

- Below are the List of popular and commonly used events.

  1. Mouse Events
    1. click:- Triggered when an element is clicked.
    2. dblclick:- Triggered on a double-click.
    3. mouseover:- Triggered when the mouse pointer enters an element.
    4. mouseout:- Triggered when the mouse pointer leaves an element.
    5. mousemove:- Triggered when the mouse moves over an element.
    6. mousedown:- Triggered when a mouse button is pressed.
    7. mouseup:- Triggered when a mouse button is released.

  2. Keyboard Events
    1. keydown:- Triggered when a key is pressed down.
    2. keyup:- Triggered when a key is released.
    3. keypress:- Triggered when a key is pressed and released (deprecated, prefer keydown).

  3. Form Events
    1. submit:- Triggered when a form is submitted.
    2. change:- Triggered when an input element's value changes (e.g., <.input>, <.select>).
    3. input:- Triggered when an input element's value is modified (real-time).
    4. focus:- Triggered when an element gains focus.
    5. blur:- Triggered when an element loses focus.

  4. Window/Document Events
    1. load:- Triggered when a resource (e.g., page, image) finishes loading.
    2. resize:- Triggered when the browser window is resized.
    3. scroll:- Triggered when the user scrolls the page or an element.
    4. unload:- Triggered when the user leaves the page (less common).

  5. Touch Events
    1. touchstart:- Triggered when a finger touches the screen.
    2. touchmove:- Triggered when a finger moves on the screen.
    3. touchend:- Triggered when a finger is lifted from the screen.
